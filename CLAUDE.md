## Tech Stack

Laravel 13 (PHP 8.3) + Inertia.js + React 19 + TypeScript + Tailwind CSS 4 full-stack application. Auth is handled by Laravel Fortify (registration, login, password reset, email verification, 2FA via TOTP). UI components are Radix UI primitives wrapped as shadcn-style components.

## Commands

### Development

```bash
composer run dev        # Starts all dev processes concurrently: PHP server, queue worker, Pail log viewer, Vite
```

### Building

```bash
npm run build           # Vite production build
npm run build:ssr       # SSR build
```

### Linting & Formatting

```bash
composer run lint       # PHP: Laravel Pint (auto-fix)
composer run lint:check # PHP: Pint in test mode (no changes)
npm run lint            # JS/TS: ESLint (auto-fix)
npm run lint:check      # JS/TS: ESLint check only
npm run format          # Prettier (auto-fix)
npm run format:check    # Prettier check only
npm run types:check     # TypeScript type checking (no emit)
```

### Testing

```bash
composer run test               # Clears config, runs Pint check, then PHPUnit
php artisan test                # Run all tests
php artisan test --filter=TestName  # Run a single test
php artisan test tests/Feature/Auth/AuthenticationTest.php  # Run a specific file
```

Tests use SQLite in-memory; no external database needed.

### Setup from scratch

```bash
composer run setup   # install deps, copy .env, generate key, migrate, build assets
```

## Architecture

### Request flow

Browser â†’ Inertia middleware (`HandleInertiaRequests`) â†’ Laravel controller â†’ `Inertia::render('page-name', props)` â†’ React page component.

The single Blade template is `resources/views/app.blade.php`. All routing is PHP-side; there is no client-side router.

### Shared page props

`HandleInertiaRequests::share()` injects three props into every page: `name` (app name), `auth.user` (current user or null), and `sidebarOpen` (sidebar state from cookie). Types are declared in `resources/js/types/global.d.ts` via module augmentation of `@inertiajs/core`.

### Layout resolution (app.tsx)

Page name string determines layout at runtime:
- `welcome` â†’ no layout
- `auth/*` â†’ `AuthLayout`
- `settings/*` â†’ `[AppLayout, SettingsLayout]` (nested)
- everything else â†’ `AppLayout`

### Wayfinder (type-safe routes)

Laravel Wayfinder auto-generates TypeScript route helpers into `resources/js/routes/` and action helpers into `resources/js/actions/`. These are regenerated by Vite on route/controller changes. Import from `@/routes` or `@/actions` â€” never hard-code URLs in frontend code.

### Authentication (Fortify)

All auth routes and logic come from Laravel Fortify. `FortifyServiceProvider` wires Inertia views to Fortify's view hooks. Custom actions live in `app/Actions/Fortify/`. Enabled features: registration, password reset, email verification, 2FA (TOTP with confirmation required).

### Settings area

`settings/` routes in `routes/settings.php` are handled by `ProfileController` and `SecurityController`. Security page manages password changes and 2FA setup/teardown. 2FA setup uses a modal (`two-factor-setup-modal.tsx`) with OTP confirmation before enabling.

### Frontend component conventions

- `resources/js/components/ui/` â€” base UI primitives (Radix-based, CVA variants)
- `resources/js/components/` â€” app-specific composed components
- `resources/js/hooks/` â€” React hooks
- `resources/js/pages/` â€” Inertia page components (one file per route)
- `resources/js/layouts/` â€” layout wrappers

The `@/` alias resolves to `resources/js/`.

### PHP conventions

- Pint uses the `laravel` preset (PSR-12 + Laravel style)
- PHP 8.3 attributes used for model `#[Fillable]` and `#[Hidden]` instead of array properties
- Form requests in `app/Http/Requests/` handle validation; controllers stay thin
- Validation rule traits in `app/Concerns/` (`PasswordValidationRules`, `ProfileValidationRules`)
